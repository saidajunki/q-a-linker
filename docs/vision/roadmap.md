# QALinker ロードマップ

## フェーズ0：MVP（現在）

### 目標
ローカル環境で動作する最小限のプロダクトを構築し、コアコンセプトを検証する。

### スコープ

#### 実装すること
- 質問投稿（チャットUI）
- AIによる質問の内部構造化（カテゴリ/レベル/意図/不足情報）
- 回答者マッチング＋Push通知（対象複数人）
- 回答投稿（チャットUI）
- AIによる回答の初心者向け翻訳/要約
- 複数回答の統合（統合回答の生成）
- 軽い評価（ありがとう/役に立った）
- 通報、最低限のモデレーション
- ローカル開発環境（Expo + NestJS + DB）

#### 意図的にやらないこと
- 送金・課金・報酬の現金化
- 公開掲示板（検索・タイムライン）
- AIの単独回答
- ランキング・競争要素

### 技術スタック
- フロント：React Native Expo
- バックエンド：NestJS
- DB：PostgreSQL（Docker）
- AI：OpenAI/Claude API（環境変数で切替）またはモック

### 成功指標
- 質問投稿→初回回答までの時間（中央値）
- 質問者の満足（ありがとう率）
- 回答者の継続率（週次）
- 通報率 / ブロック率（安全性）

## フェーズ1：クローズドβ

### 目標
限定ユーザーでの実運用を通じて、プロダクトを磨き込む。

### 追加機能
- SNS認証（Google/Apple）
- 回答者の得意「レベル帯」の手動設定
- 質問者への聞き返し機能（不足情報の補完）
- 回答の編集履歴
- より詳細なモデレーション機能

### インフラ
- AWS ECS Fargate
- RDS PostgreSQL
- SQS（非同期処理）

---

## フェーズ2：オープンβ / 一般公開

### 目標
ユーザー基盤を拡大し、ネットワーク効果を発生させる。

### 追加機能
- カテゴリの拡充
- 回答者の得意「レベル帯」の自動学習
- バッジシステム（初回回答、継続回答、ありがとう獲得数）
- 音声入力（質問）

### マーケティング
- 特定カテゴリ（例：プログラミング初心者）に絞った集客
- 回答者コミュニティの育成

---

## フェーズ3：収益化開始

### 目標
持続可能なビジネスモデルを確立する。

### 追加機能
- 有償回答枠（SLA付き）
- 質問者向けサブスクリプション
- 統合回答への貢献度に応じた報酬配分
- ポイントシステム

### ビジネス
- 回答者への報酬支払い機能
- プラットフォーム手数料の設定

---

## フェーズ4：B2B展開

### 目標
法人顧客を獲得し、収益を多角化する。

### 追加機能
- 企業向け管理画面
- SSO連携
- カスタムカテゴリ設定
- 分析ダッシュボード

### ユースケース
- 社内Q&Aシステム
- カスタマーサポートの前段階

---

## フェーズ5：ナレッジ資産化

### 目標
蓄積データを資産として活用する。

### 追加機能
- 質問の公開・検索機能
- FAQ自動生成
- ナレッジベースAPI

### ビジネス
- ナレッジベースのライセンス販売
- RAG用データセット提供

---

## フェーズ6：AIファクトチェック層

### 目標
AI時代のインフラとしてのポジションを確立する。

### 追加機能
- MCP（Model Context Protocol）対応
- 外部AI回答の検証API
- 「人間検証済み」ラベル

### ビジネス
- 検証リクエストの従量課金
- SLA付きプラン

---

## 判断基準：次のフェーズに進む条件

各フェーズから次に進む際の判断基準：

| フェーズ | 次に進む条件 |
|---------|-------------|
| MVP → クローズドβ | コア機能が安定動作、初期ユーザーからのフィードバック収集完了 |
| クローズドβ → オープンβ | 主要なバグ修正完了、回答者の継続率が目標達成 |
| オープンβ → 収益化 | MAU目標達成、質問→回答のサイクルが安定 |
| 収益化 → B2B | 個人向け収益が安定、法人からの引き合いあり |
| B2B → ナレッジ資産化 | 十分なQ&Aデータ蓄積、データ品質の担保 |
| ナレッジ資産化 → AIファクトチェック | MCP標準の普及、AI検証ニーズの顕在化 |
