# QALinker プロジェクトルール

このプロジェクトの開発ルールは `rules/` ディレクトリに記載されています。
開発時は以下のファイルを必ず参照してください。

## 必須参照

- `rules/always.md` - 常に遵守すべきルール・優先順位
- `rules/README.md` - ルール一覧

## ビジョンドキュメント

設計判断に迷った際は、以下のドキュメントを参照すること：

- `docs/vision/philosophy.md` - 設計哲学と判断指針
- `docs/vision/business-model.md` - ビジネスモデル
- `docs/vision/roadmap.md` - フェーズ別計画
- `docs/vision/knowledge-architecture.md` - ナレッジベース設計

特に `philosophy.md` の「設計判断の指針」セクションは、機能追加や設計判断の際に必ず確認すること。
